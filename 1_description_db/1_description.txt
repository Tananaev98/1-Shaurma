Таблицы:

1) users содержит логины, пароли, Фио, дату рождения и прочую информацию о сотрудниках
	
	Структура:
	id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение
	
	login формат: varchar(50); назначение: логин для авторизации; параметры: уникальное значение
	
	password: varchar(50), назначение: пароль для авторизации; параметры: не может быть пустым
	
	name: varchar(50), назначение: имя пользователя; параметры: не может быть пустым 
	
	family varchar(50), назначение: фамилия пользователя; параметры: не может быть пустым
	
	middleName varchar(50), назначение: отчество пользователя; параметры: не может быть пустым
	
	birthDate date, назначение: дата рождения пользователя; параметры: не может быть пустым
	
	role varchar(50), назначение: должность пользователя для аутентификации; параметры: не может быть пустым
	
	
	
2) 	menu таблица хранящая в себе объекты меню

	Структура:
	
	id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение
	
	name формат: varchar50; назначение: наименование объекта меню; параметры: не может быть пустым 	
	
	price формат: int; назначение: цена товара; параметры: не может быть пустым

	description формат: varchar150; назначение: описание товара; параметры: не может быть пустым
	
	link_image формат: varchar50; назначение: ссылка на изображение в катологе сервера; параметры: Уникальное значение
	
	
	
	1) Для начала создаем таблицу объектов меню в бд с полями идентификатор, наименование, цена, описание, ссылка на картику в каталоге.
	
	
	

3) reserves Таблица которая хранит в себе шаблоны объектов "Запасы" на основаниии этих объектов создаются поступления товаров. 

	Структура:
	
	id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение
	
	name формат: varchar50; назначение: наименование объекта меню; параметры: не может быть пустым 	
	
	min_quantity формат: int; назначение: минимальное количество в котором товар может поступать; параметры: не может быть пустым
	
	max_quantity формат: int; назначение: максимальное количество в котором товар может поступать; параметры: не может быть пустым
	
	critical_quantity формат: int; назначение: Критическое количество товара при котором директор должен быть уведомлен; параметры: не может быть пустым

    availability формат: varchar50; назначение: доступен ли объект для использования(создания на основании его поступлений); параметры: по умолчанию должен быть в значении "Не доступен" 	




4) remains_reserves Таблица которая хранит в себе "Поступления товаров" который создаются на основании данных из таблицы reserves

	Структура:
	
	id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение
	
	id_reserves формат: int; назначение: внешний ключ; параметры: не может быть пустым
	
	date_of_receipt формат: date; назначение: дата поступления товара; параметры: не может быть пустым 	
	
	cost_gram формат: int; назначение: стоимость товара за один грам; параметры: не может быть пустым
	
	quant_received формат: int; назначение: количество в котором товар поступил; параметры: не может быть пустым
	
	quant_spent формат: int; назначение: количество товара в котором он израсходован; параметры: не может быть пустым

	expiration_date формат: date; назначение: дата истечения срока годности; параметры: не может быть пустым 	

    status формат: varchar50; назначение: Пригоден для использования, просрочен, израсходован, срок истекает, не доступен; параметры: по умолчанию должен быть в значении "Не доступен" 
	
	
	
	
5) requirement_reserves Таблица которая хранит в себе список и количество ингредиентов необходимых для приготовления каждого объекта меню

	Структура:
	
	id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение
	
	id_menu формат: int; назначение: внешний ключ; параметры: не может быть пустым
	
	id_reserves формат: int; назначение: внешний ключ; параметры: не может быть пустым
	
	quant_reserv формат: int; назначение: Количество; параметры: не может быть пустым
	
	

Нужна таблица "Заказы" когда пользователь добавляет свой первый товар в корзину создается запись в таблице заказов со статусом "Не оплачен"
содержимое заказа с таким статусом отображается в корзине. 





6) Orders Таблица которая хранит в себе список заказов

	Структура:
	
	id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение
	
	datetime_order формат: datetime; назначение: дата и время оформления заказа; 
	
    status формат: varchar50; назначение: Создан, оплачен, в работе, завершен параметры: Создан индекс уникальности для обеспечения того что заказ со статусом "Создан" может быть только один, это нужно для обеспечения логики программы.
	






7) Orders_menu соединительная таблица между заказами и объектами меню

id формат: int; назначение: первичный ключ; параметры: уникальное значение, авто увеличение

id_order формат: int; назначение: внешний ключ к таблице заказов; параметры:

id_menu формат: int; назначение: внешний ключ к таблице меню; параметры: 

quant_reserv формат: int; назначение: Количество в котором пользователь заказал товар в рамках этого заказа; параметры: не может быть пустым
